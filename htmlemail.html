HTML Email

HMTL5 CSS3 Sass PHP JavaScript JQuery

A webpage for sending a HTML Email, which supports user input including image upload and text fields. The user input is put into the HTML Email template.

This was a project for the Professional Portfolio module which was set by Grooup.

Requirements

Requirements consisted of:

  • Must work on all email clients
  • Must contain editorial cntent
    to update the customers of
    grooup.com on news and projects
  • must follow the style of grooup.com
  • Must be responsive
  • Must be mobile first
  • Must be hand coded code
  • Must be tested against all email clients
  • Must be stored in version control
  • Must be commented and well formatted
  • Must use semantic HTML
  • Layout must work across all devices
  • Must use Sass
  • Must support IE8+

In addition to these, the following requirements were identified for the webpage sending the email:

  • The user must be able to send the email to a custom
    email address
  • The email must automatically generate a personalised
    greeting
  • The user must be able to specify the subject line of the email
  • The user must be able to add a signature
  • The user must not have to type HTML
  • The user must be able to include images
  • The user must be able to specify alt text for images
  • The user must be able to write content about a project
  • The user must be able to include multiple projects
  • The webpage must store images
  • The webpage must ensure only images can be uploaded
  • The webpage must not overwrite duplicated images
  • The webpage must create directories for storing images
  • The weboage must replace spaces in the file name
  • The webpage must display the email that was sent

Design

As the email had to be in the style of grooup.com , sketches and mockups were not used, as there was already a webpage with the design implemented. Sketches were used for the webpage, but as the design was being kept simple, mockups were not used.

Development

The panel was developed using PHP and an object oriented approach, with the mailer object being used to process user input and all aspects of the email, and the project object being used for each project, to check, rename and store images, as well as storing the projects title, image alt text, text and link.

While the styling of the email was created using Sass, the compiled CSS had to be placed in the <head> section, so it could be inlined using htmlemail.io. Using an external stylesheet or leaving the styles in the <head> section meant they were not applied to the email, and only inline styling would work correctly.

Testing

The PHP webpage was tested to ensure that it functioned as expected, which consisted of: ensuring the user input was passed to the script, testing the script could send the email, ensuring the script could assemble the email correctly (placing user input in the correct places), ensuring images were stores, not overwritten and spaces were replaced with underscores and ensuring the sent email was displayed for the user to see.

The HTML email was tested to ensure that: images were displayed, the layout was implemented correctly and that the email rendered correctly in multiple clients. Testing the email rendered correctly in clients was completed using emailonacid.com.

From the mobile test results, it can be seen that images were not rendered on one device, however, content was rendered appropriately on all other devices

From the desktop results, it can be seen that the email rendered incorrectly on nine devices, which made up the majority.

From the web test results, it can be seen that the email rendered correctly on all devices

In conclusion, whilst it is fairly easy to design for mobile or desktop, it is quite difficult to master both. This email template was designed using percentages and mobile first. When it came to desktop, all web based clients rendered the email correctly, however it was actual desktop software that could not.
