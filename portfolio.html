Portfolio Website

HMTL5 CSS3 Sass PHP JavaScript JQuery

This website was built to showcase me and my work, including a contact form which sends user input via email.

Requirements

  • Must be stored in version control
  • Must be commented and well formatted
  • Must use semantic HMTL
  • Layout must work across devices
  • Must support IE8+
  • Must use Sass
  • Must have a logo
  • Must have calls to action
  • Must contain an ‘about me’ section
  • Must contain a projects section
  • Must have a contact form
  • Must contain social media icons
  • Must display my education
  • Must display my skills
  • Must contain a link to download my CV

Design

Both sketches and mockups were created for this website, ensuring that ideas were clearly documented and gradual improvement could be seen. This website was designed for 5 device sizes: mobile, tablet (portrait), tablet (landscape)/small laptop, regular laptop and large desktop.

The following image shows multiple sketches which were not chosen for the final design, but demonstrate the process (I'll remember to use a sharper pencil next time).

Development

The development of this site mainly focussed on the Sass, as this makes up the majority of the code. This website uses a mobile first approach and adapts the styles depending on the device size.

The design was added to during development, as better ways of displaying content were identified, such as the dropdown boxes that contain education information, and hover styles on the skills that display code in the pages background.

The JavaScript is not object oriented, as JavaScript usage was minimal and therefore would have been excessive for this website.

The animations on the chevron and the logo were implemented last as they were suggested towards the end of the development of this project.

Testing

The functionality of this website was tested during development to ensure that the features function correctly. As new features were added, some others were broken, but repaired along the way. Integration testing may be of use in future.
