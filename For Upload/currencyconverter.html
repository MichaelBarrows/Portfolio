Currency Converter

HMTL5 CSS3 Sass API JavaScript JQuery

Summary

A currency converter widget that gets the latest rates using an API, and converts from GBP to the five most popular currencies.

Try me!

Requirements

Requirements consisted of:

  • Must be self-contained
  • Must get the latest exchange rates
  • Must support multiple currencies
  • Must validate user input
  • Must be responsive
  • Must use all client-side code
  • Must be stored in version control
  • Must be commented and well formatted
  • Must use semantic HTML
  • Must use Sass
  • Must support IE8+

Design

8 sketches were created, 4 for the standalone version, 3 for the integrated version and 1 page sketch to show the position of the integrated element in comparison to the rest of the page.

Mockups were not used as the sketches were detailed enough to start building from.

Development

The currency converter was developed as both a standalone and integrated solution, the website given as the example for integration was doggiegadgets.com. The integrated version was created by making a copy of a page on the website and integrating the currency converter into an existing form, with JavaScript used to change the original product prices to the new prices.

For development, the currency converter uses an object-oriented approach, and the API fixer.io, which was used for obtaining the exchange rates. 5 currencies are used within the converter, the US Dollar, the Euro, the Japenese Yen, the Australian Dollar and the Canadian Dollar as these were found to be the most popular.

Testing

The currency converter testing consisted of manually checking calculations for all currencies, checking text and no input was handles with an error message and ensuring the functionality of the application performed as expected.

The application will accept text after numbers, but will disregard anything after the numbers to ensure the application can still function

The testing revealed that the application functioned perfectly on desktop, however on mobile, the flags on the currency select failed to update until the select was clicked again, implying an issue with the JQuery event handler for mobile selects.
